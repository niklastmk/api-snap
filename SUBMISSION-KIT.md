### 2. Generate example API responses

Open a terminal and run these to save example responses (replace `YOUR_KEY` with your API key from the dashboard):

```bash
# QR Code
curl -s "https://api-snap.com/api/qrcode?data=https://api-snap.com&api_key=YOUR_KEY" -o qr-example.png

# Screenshot
curl -s "https://api-snap.com/api/screenshot?url=https://api-snap.com&api_key=YOUR_KEY" -o screenshot-example.png

# Hash
curl -s "https://api-snap.com/api/hash?text=hello+world&api_key=YOUR_KEY" | python3 -m json.tool > hash-example.json
```

## RapidAPI Hub (~45 min)

**What it is:** The largest API marketplace. Millions of developers search RapidAPI to discover and subscribe to APIs. Your listing gets its own page with interactive testing, auto-generated code snippets, and a pricing table.

**Why it matters:** RapidAPI proxies all requests, handles billing, and exposes your API to their entire user base. Developers can subscribe and start calling your API without ever visiting api-snap.com.

### Account Setup

1. Go to **https://rapidapi.com/auth/sign-up** — sign up (GitHub login works)
2. Go to **https://rapidapi.com/studio** — this is the provider dashboard

### Create Your API Listing

1. In RapidAPI Studio, click **"Add New API"**
2. Fill in:

| Field | Value |
|-------|-------|
| API Name | API Snap |
| Short Description | 13+ utility APIs behind one key — QR codes, screenshots, PDFs, image processing & more |
| Category | **Tools** |
| Upload logo | Use your 512×512 PNG |

3. Click **"Add API"**

### Import the OpenAPI Spec

1. Go to the **"Definition"** tab of your new API
2. Click **"Import"** → select **"OpenAPI"**
3. Choose **"Import via URL"** and paste: `https://api-snap.com/openapi.json`
4. Click **"Import"**
5. Review the imported endpoints — all 13+ should appear

### Configure the Base URL

1. Go to **"Configuration"** → **"Base URL"**
2. Set base URL to: `https://api-snap.com`
3. RapidAPI will proxy requests through `{api-name}.p.rapidapi.com` → your base URL

### Set Up Authentication

1. In **"Configuration"** → **"Security"**:
   - RapidAPI injects its own `X-RapidAPI-Key` header
   - You need to map this to your auth: Under **"Runtime Auth"**, configure it to pass the API key as a `Bearer` token in the `Authorization` header
   - **Alternatively:** Create a dedicated API key in your dashboard specifically for RapidAPI's proxy — set it as a static header that RapidAPI injects on every request

**Recommended approach (simplest):**
- Create a new user account on API Snap (e.g., `rapidapi-proxy@api-snap.com`)
- Generate an API key for it
- In RapidAPI Configuration → "Headers" → add a static header:
  - Header: `Authorization`
  - Value: `Bearer YOUR_RAPIDAPI_PROXY_KEY`
- Set the account to Business tier (100K calls) or create a special unlimited tier

### Configure Pricing Plans

Go to **"Plans & Pricing"** tab and create these plans:

| Plan | Monthly Price | Rate Limit | Quota |
|------|--------------|------------|-------|
| Basic (free) | $0.00 | 10 requests/minute | 100 calls/month |
| Hobby | $9.00 | 30 requests/minute | 1,000 calls/month |
| Pro | $29.00 | 60 requests/minute | 10,000 calls/month |
| Business | $99.00 | 120 requests/minute | 100,000 calls/month |

**Note:** RapidAPI takes a 20% revenue share on paid subscriptions. Your $9 plan nets $7.20, etc. Adjust if needed, or keep prices the same for simplicity.

### Add Example Responses

This is critical for discoverability — listings with examples rank higher.

1. Go to the **"Endpoints"** tab
2. For each key endpoint (QR, Screenshot, Hash, UUID, Lorem):
   - Click the endpoint → **"Example Responses"**
   - Click **"Add Example"**
   - Paste the JSON response from the examples you saved earlier
   - Add a descriptive name (e.g., "Generate QR code for URL")
3. Prioritize: QR Code, Screenshot, and Hash — these are the most eye-catching

## Postman Public API Network (~30 min)

**What it is:** Postman's public directory of API collections. Publishing here lets millions of Postman users discover, fork, and test API Snap directly in Postman.

**Why it matters:** Postman users import collections with one click and start testing immediately — zero friction. Your collection shows up in Postman's search and the public API Network directory.

### Account Setup

1. Go to **https://www.postman.com/postman-account/** — sign up or log in
2. Create a **Team Workspace** named **"API Snap"** (free plan is fine)
   - Go to Workspaces → "Create Workspace" → name it "API Snap" → set visibility to **Public**

### Import the OpenAPI Spec

1. Inside your "API Snap" workspace, click **"Import"** (top left)
2. Choose **"Link"** and paste: `https://api-snap.com/openapi.json`
3. Click **"Import"**
4. Postman will create:
   - An **API definition** (the spec)
   - A **Collection** (callable endpoints)

### Configure the Collection

1. Open the imported collection (left sidebar → "Collections" → "API Snap")
2. Click the collection name → **"Variables"** tab
3. Add/edit these variables:

| Variable | Initial Value | Current Value |
|----------|--------------|---------------|
| `baseUrl` | `https://api-snap.com` | `https://api-snap.com` |
| `apiKey` | *(leave blank — users fill in their own)* | *(blank)* |

4. Go to **"Authorization"** tab on the collection:
   - Type: **Bearer Token**
   - Token: `{{apiKey}}`
   - This applies auth to all requests in the collection

### Add Example Responses

For each key endpoint (do at least QR, Screenshot, Hash, UUID, Lorem):

1. Click the endpoint in the collection
2. Click **"Send"** (with your own API key set temporarily)
3. In the response pane, click **"Save as Example"** (the disk icon, top right of response)

## Common Pitfalls to Avoid

1. **Don't launch on a Monday or Friday** — lowest traffic days
2. **Don't ask for upvotes** — PH detects and penalizes this (including from the same IP/network)
3. **Don't launch until RapidAPI/Postman are live** — PH visitors will check your credibility
4. **Don't argue with critical comments** — thank them, learn from it
5. **Don't launch at a random time** — midnight PT is when the clock resets, giving you the full 24-hour window
